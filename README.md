# Dry Bean 干豆品种多分类算法对比实验
> AIT209 机器学习与项目实践 期末课程作业

本项目基于干豆形态学特征数据集，完成了从数据清洗、特征工程到多分类算法对比的全流程实验。项目采用工程化模块架构，实现了5种分属不同范式的多分类算法，从分类精度、收敛特性、过拟合程度、噪声鲁棒性多个维度开展量化对比，为干豆品种自动识别任务提供完整的算法选型依据。

## 一、数据集介绍
本实验使用 **Dry Bean Dataset** 干豆分类数据集，包含7种常见干豆品种（SEKER、BARBUNYA、BOMBAY、CALI、DERMASON、HOROZ、SIRA）的16项计算机视觉提取的形态学数值特征。

### 数据集划分
数据集按照 7:2:1 的比例划分为训练集、验证集、测试集，划分情况如下：

| 数据集 | 样本数量 | 特征数量 | 占比 |
|--------|----------|----------|------|
| 训练集 | 9527 | 16 | 70.00% |
| 验证集 | 1347 | 16 | 9.89% |
| 测试集 | 2737 | 16 | 20.11% |

### 数据预处理
针对原始数据存在的标签错误、特征缺失、异常值、多重共线性、类别不平衡五大问题，完成了全流程清洗与特征工程：
1.  **标签规范化**：修正大小写、字符替换、前后空格等18种标签错误，统一为7个标准类别
2.  **缺失值填充**：基于训练集均值填充Perimeter、Solidity两个特征的随机缺失
3.  **异常值处理**：采用1%/99%分位数截断法处理极端异常值
4.  **特征筛选**：通过方差过滤与相关性剔除，精简高冗余特征
5.  **特征标准化**：使用StandardScaler统一特征量纲为均值0、方差1
6.  **类别均衡**：训练集采用SMOTE过采样，平衡不同类别样本数量

## 三、实现算法
项目共实现5种多分类算法，覆盖线性分类、核方法、惰性学习、深度学习、集成学习五大范式，其中XGBoost为课程内容之外自主查阅资料实现的拓展算法。

1.  **逻辑回归 (Logistic Regression)**：线性分类基准模型，采用L2正则化与lbfgs优化器，可解释性强，作为性能基线。
2.  **支持向量机 (SVM)**：采用RBF高斯核，通过核技巧映射高维空间寻找最优分类超平面，擅长处理非线性特征。
3.  **K近邻 (KNN)**：惰性学习算法，设置K=5，基于欧氏距离投票分类，训练成本低。
4.  **全连接神经网络 (ANN)**：两层隐藏层（64+32神经元），ReLU激活+Adam优化器，通过反向传播拟合非线性映射。
5.  **XGBoost（课外拓展实现）**：极端梯度提升树集成算法，串行训练多棵决策树拟合残差，是表格数据分类的工业界主流方案。

## 四、核心实验结果
所有实验基于Clean干净数据集，固定随机种子42保证可复现，在测试集上的核心分类指标如下：

| 算法 | 准确率 | 精确率 | 召回率 | F1值 |
|------|--------|--------|--------|------|
| 逻辑回归 | 0.9043 | 0.9055 | 0.9043 | 0.9045 |
| SVM | 0.9127 | 0.9130 | 0.9127 | 0.9128 |
| KNN | 0.8951 | 0.8956 | 0.8951 | 0.8953 |
| ANN | 0.9123 | 0.9129 | 0.9123 | 0.9122 |
| XGBoost | 0.9068 | 0.9072 | 0.9068 | 0.9069 |

### 核心结论
1.  **分类精度**：SVM与ANN表现最优，测试集准确率均超过91%，非线性模型效果显著优于线性基准。
2.  **泛化能力**：SVM过拟合程度最低，训练集与测试集精度差仅2.05%；KNN与XGBoost存在中度过拟合。
3.  **噪声鲁棒性**：XGBoost相对下降幅度最小，鲁棒性最优；SVM在所有噪声强度下绝对精度最高，综合稳定性最佳。
4.  **收敛特性**：ANN损失下降速度快，20轮左右进入收敛区间；XGBoost下降平稳，60轮左右完全收敛。

完整的实验图表与深度分析见项目报告与 `results/plots/` 目录。

## 五、环境与运行方式
### 依赖环境
项目基于Python开发，核心依赖如下：
- Python 3.8+
- numpy、pandas（数据处理）
- scikit-learn（基础算法、指标计算、预处理）
- xgboost（梯度提升树算法）
- matplotlib、seaborn（可视化绘图）
- pyyaml（配置文件解析）
- joblib（模型持久化）

一键安装依赖：
```bash
pip install -r requirements.txt
```

### 运行方式
项目支持命令行一键运行，无需UI界面。在项目根目录执行：
```bash
python main.py
```

可选命令行参数：
- `--config`：指定配置文件路径，默认 `configs/default.yaml`
- `--algorithms`：指定运行的算法，逗号分隔，默认运行全部5种
- `--no-plot`：跳过绘图输出
- `--no-analysis`：跳过深度分析（过拟合、鲁棒性）
- `--output-dir`：指定结果输出目录，默认 `results/`

示例：仅运行SVM和XGBoost并输出到指定目录
```bash
python main.py --algorithms svm,xgboost --output-dir ./results/exp1
```

运行完成后，所有指标表格、分析报告、可视化图表将自动输出到结果目录。
